# Contributing to VertexSDR

Bug reports, testing, documentation, and code contributions are all welcome.

## Reporting Bugs

Open an issue on GitHub: https://github.com/Steven9101/VertexSDR/issues

Include:
- OS and hardware
- Build flags (e.g. `make USE_VULKAN=1`)
- Config file (remove any private info)
- Full console/log output
- Steps to reproduce

## Feature Requests

Open an issue. Describe what you want and why it would be useful.

## Code Contributions

1. Open an issue first to discuss the change
2. Fork, create a branch, make the change
3. Make sure it builds: `make clean && make`
4. Open a PR with a clear description

## Code Style

- C99
- No comments except copyright headers
- Names should explain the code
- Match the existing style

## Hardware Testing

If you test VertexSDR on real hardware, open an issue with:
- Device name and model
- Build flags used
- Sample rates tested
- Whether it worked
- Any issues found
